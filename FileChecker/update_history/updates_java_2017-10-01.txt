SUMMARY:
========

CONFIG_MISSION_NUMBER checks:
- added to meta-data, profile, and trajectory (udpated) file checks
- a value of 0 is not allowed anywhere
- FillValue:
  - allowed in R-mode profile files and R-mode cycles in trajectory files
  - allowed with WARNING in meta-data file

- ***TEMPORARILY*** zero values only generates warnings to allow DACs to adjust

Reference Table input:
- detects a status of "deprecated" - marks the entry as such
- detects the "active" statuses - marks the entry as valid
- detects other statuses - marks the entry as invalid

Reference Table checks:
- "deprecated" entries generate WARNING (but are accepted)
- "invalid" entries generate ERRORS and are rejected
